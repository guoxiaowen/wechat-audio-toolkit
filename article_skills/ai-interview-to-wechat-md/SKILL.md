# AI 外网访谈 → 公众号 Markdown 文章

## 背景与定位

这个 Skill 用于将外网（YouTube、播客、Twitter Spaces 等）的 AI 领域访谈内容转化为中文公众号文章。

**内容来源：** 外网比较火的 AI 相关最新访谈，嘉宾可能是 AI 公司创始人、研究者、投资人、产品负责人等。逐字稿可能是英文原文，也可能是已有的中文翻译。

**读者群体：** 主要面向关注 AI 带来的商业机会、想用 AI 提升效率或寻找新项目的创业者和商业人群。但具体到每篇文章，目标读者会因访谈内容不同而不同——这需要你自己分析判断，不能套用固定人群。

**为什么用 Q&A 结构：** 访谈天然是问答形式。以主持人/采访者的核心问题为骨架来组织内容，比按主题重新归类更真实、更有代入感。

## 你的任务

将访谈逐字稿转化为三样东西：

1. **目标读者分析**（这篇内容最适合谁看、为什么）
2. **5 个标题选项**（基于目标读者分析，附选择理由）
3. **1 篇完整的 Q&A 结构 Markdown 文章**（中文，可直接通过 Markdown 转换器生成公众号排版）

## 工作流程

### 重要原则：宁长勿短，不要压缩

你的默认倾向会是压缩和合并内容，但在这个任务里，这个倾向是错的。

一场 1-2 小时的访谈，整理后的文章通常在 **6000-10000 字**之间。如果你的输出只有 3000-4000 字，几乎可以肯定你过度压缩了。

读者来看这篇文章，是想了解 AI 领域最前沿的人在想什么、在做什么、看到了什么趋势。他们需要的是有深度的洞察和具体的判断，不是一个浮光掠影的概览。采访者花时间展开讲的每个话题，都有其编辑价值。你的任务是忠实呈现这些内容，不是替读者做"精简"的决定。

### 第一步：通读全文，建立全局理解

仔细阅读完整的逐字稿。在这一步你需要搞清楚：

1. **谁在说话：** 识别采访者和受访者的身份。搞清楚每个人是谁、在哪家公司/机构、什么角色。
2. **受访者是什么人：** 职位、公司、核心成就、为什么这个人的观点值得关注。
3. **访谈的主线：** 覆盖了哪些话题？从哪里聊到哪里？
4. **语言处理：** 如果是英文逐字稿，你需要在后续步骤中翻译成中文。翻译原则见下方。

### 第二步：目标读者分析（最关键的新增步骤）

这一步是这个 Skill 的核心差异点。你没有预设的目标人群，必须自己从内容中推导出来。

**这一步不能省略，不能敷衍，不能用一两句话带过。** 它直接决定了标题策略和内容侧重的质量。请认真思考并输出完整的分析。

分析维度：

**a) 这篇访谈的核心信息是什么？**
不是"聊了 AI"这种泛泛的总结，而是具体到：它揭示了什么趋势？提出了什么判断？分享了什么方法？透露了什么信号？

**b) 谁最需要知道这些信息？为什么？**
想象一下：这篇内容发出去之后，谁看完会觉得"这篇文章太及时了"？是正在做 AI 应用的创业者？是考虑转型的传统行业老板？是想了解 AI 对自己工作影响的职场人？是关注投资机会的人？

列出 2-3 个最匹配的细分人群，每个人群写一句话说明为什么他们需要这篇内容。

**c) 这些人群在什么场景下会点开这篇文章？**
他们在焦虑什么？在好奇什么？在寻找什么？这决定了标题应该打哪个情绪点。

**输出格式：**

```
## 目标读者分析

**核心信息：** [一两句话概括这篇访谈最有价值的信息]

**最匹配的读者群体：**
1. [人群1]：[为什么他们需要这篇内容]
2. [人群2]：[为什么他们需要这篇内容]
3. [人群3]：[为什么他们需要这篇内容]

**情绪切入点：** [这些读者在什么心理状态下会被吸引点开？]
```

### 第三步：提取核心问题，构建 Q&A 骨架

从对话中提炼出采访者提出的核心问题，这些问题就是文章的骨架。

**提取原则：**

- **保留提问者的身份：** 标注是谁问的
- **合并碎片化的追问：** 把分散的问题合并成一个完整、清晰的问题
- **保留追问的层次感：** 深挖同一话题的追问放在同一章节里
- **过滤闲聊和广告：** 节目赞助商广告、不相关的闲聊不需要保留

**分章逻辑——核心原则：采访者分开讲的话题，就必须分成独立章节。不要合并。**

具体来说：
- 如果采访者花了几分钟做背景铺垫或概念解释，这就是一个独立章节
- 如果采访者分别用不同的问题去聊技术趋势、商业模式、竞争格局、未来预判，这些就是各自独立的章节
- 如果访谈中有一段关于个人经历/创业故事的讨论，也应该是独立章节

判断标准：**采访者是不是换了一个新的问题方向？** 如果是，那就是一个新章节。

一场典型的 1-2 小时访谈，整理后通常会有 **8-12 个章节**。如果你只分出了 5-6 个章节，很可能是合并过度了。

### 第三步半：先输出大纲，再逐章展开

在正式写文章之前，先把 Q&A 骨架以大纲形式列出来：

```
章节一：[标题]
  - 🎙 [采访者]：[问题1概要]
  - 🎙 [采访者]：[问题2概要]

章节二：[标题]
  ...
```

列完大纲后自检：
- 章节数量是否在 8-12 个之间？
- 有没有把本应独立的话题合并了？
- 采访者专门做的背景铺垫有没有遗漏？

大纲确认无误后，再逐章展开。

### 第四步：填充回答内容

在每个问题下面，整理受访者的回答。

**整理原则：**

- **保留思考的质感：** 受访者如果用了一个很精妙的比喻或表达方式，翻译时要保留其精髓，不要抹平成平淡的陈述
- **去掉冗余但不去掉深度：** 去掉无意义的重复和口头禅，但保留论证的逻辑链和具体的例子
- **压缩但不失真：** 可以压缩篇幅，但核心观点、论据和数据不能丢
- **数据必须准确：** 所有数字、公司名、产品名、时间线都要准确
- **专业术语处理：** AI 领域的专业术语首次出现时用「中文（英文原文）」的格式，如「大语言模型（LLM）」「检索增强生成（RAG）」。后续出现可以只用中文或大家熟知的英文缩写

**金句提取：** 受访者特别有洞察力、判断力的话，用引用块单独呈现。AI 领域的访谈往往有很多这样的金句——关于趋势的判断、关于技术的比喻、关于竞争的预测——这些是读者最想看的内容。

### 第五步：翻译原则（如果原文是英文）

如果逐字稿是英文，翻译时遵循以下原则：

- **信息准确优先：** 不要为了中文通顺而改变原意
- **口语感保留：** 访谈的口语感是这类内容的魅力，翻译不要过度书面化
- **不要翻译专有名词：** 公司名（OpenAI、Anthropic、Google）、产品名（ChatGPT、Claude、Gemini）、人名保留英文。技术术语保留英文缩写（LLM、RAG、AGI、API）
- **长句拆短：** 英文的长从句翻译成中文时要拆成短句，符合中文阅读习惯
- **文化适配：** 如果受访者引用了中国读者不熟悉的文化典故或案例，可以加一句简短的背景说明（用括号）

### 第六步：撰写嘉宾介绍卡片

在文章开头写一段受访者介绍（1-2句话），然后用表格呈现关键信息。

对于 AI 领域的受访者，关键数据可能包括：
- 当前职位/公司
- 过往履历中最亮眼的一条
- 公司/产品的核心数据（用户量、融资额、估值等，如果在访谈中提到了的话）
- 访谈日期/来源

### 第七步：生成 5 个标题选项

标题是这篇文章能否获得阅读的关键。

**你的标题策略必须基于第二步的目标读者分析。** 不要跳过那一步直接写标题——没有清晰的读者画像，标题一定是泛泛的、没有穿透力的。

**标题要素——每个标题包含以下要素中的至少3个：**

| 要素 | 作用 | 示例 |
|-----|------|------|
| 人物身份/权威性 | 让读者觉得"这个人说的话值得听" | OpenAI CTO、前 Google AI 负责人 |
| 核心观点/判断 | 制造"我必须知道"的紧迫感 | "AGI 两年内到来"、"90%的 AI 公司会死" |
| 行业/领域 | 让目标读者精准识别"这跟我有关" | AI Agent、大模型、具身智能 |
| 数据/时间线 | 增加可信度和紧迫感 | 100亿估值、6个月内、用户破亿 |
| 反常识/争议 | 制造好奇心 | "我们不训练更大的模型了"、"Scaling Law 已死" |
| 实操/方法 | 暗示有可以带走的干货 | "怎么用 AI 替代80%的工作"、"创业者该怎么选模型" |

**5 个标题覆盖不同的切入角度，基于你在第二步识别的不同读者群体：**

1. **权威观点型：** 突出受访者身份 + 最震撼的判断
2. **趋势洞察型：** 突出行业趋势，让关注方向的人点进来
3. **实操价值型：** 暗示文章有可以带走的方法论或决策依据
4. **争议/反常识型：** 用反常识的观点制造好奇心
5. **焦虑/紧迫型：** 从读者的焦虑出发（怕被淘汰、怕错过机会）

每个标题附一句话说明：这个标题打的是第二步分析中的哪个人群、利用的是什么情绪。

### 第八步：组装 Markdown 文章

按以下模板组装最终文章：

```markdown
# [正式标题——从5个选项中选一个占位，用户会自己替换]

**[受访者名] x [采访者名]**
**访谈实录 | 核心观点提炼**

---

[受访者介绍段落，1-2句话]

| 关键信息 | |
| --- | --- |
| 数据项1 | 数值1 |
| 数据项2 | **数值2**（重要数据加粗） |
| ... | ... |

---

## 一、[章节标题]

*[章节副标题，斜体，说明这一节在聊什么]*

**🎙 [采访者姓名]：[完整的问题]**

> [受访者的回答，整理/翻译后的版本]
> —— [受访者姓名]

**🎙 [采访者姓名]：[追问]**

> [受访者的回答]
> —— [受访者姓名]

> **[特别有洞察力的金句，加粗]**
> —— [受访者姓名]

---

## 二、[下一个章节标题]
...

---
```

**Markdown 格式要点：**

- `#` 用于文章大标题，`##` 用于章节标题
- 问题用 `**🎙 姓名：问题**` 格式，加粗
- 回答用 `>` 引用块，末尾标注 `—— 姓名`
- 独立金句用 `>` 引用块 + `**加粗**`
- 章节间用 `---` 水平线分隔
- 关键数据表格用 Markdown 表格语法
- 需要强调的数字或关键词用 `**加粗**`
- AI 术语首次出现用「中文（English）」格式

### 第九步：输出与交付

最终输出三部分内容，保存为一个 `.md` 文件：

**Part 1：** 目标读者分析
**Part 2：** 5 个标题选项（带简短说明）
**Part 3：** 完整的 Markdown 文章

告诉用户：
1. 先看目标读者分析，确认你认同这个判断
2. 从 5 个标题里选一个（或自己组合改编）
3. 用 Markdown 转换器（如 Md2All、墨滴等）转成公众号格式
4. 复制粘贴到公众号编辑器即可发布

## 参考范例

在 `references/example.md` 中有一篇已发布并获得 2 万+阅读的访谈整理成品文章（来自另一个公众号的小红书电商访谈）。虽然主题不同，但它展示了正确的：
- 章节粒度（10 个章节，每个话题独立）
- 每个章节内问答的深度和细节量
- 整篇文章的总体长度和信息密度
- Markdown 排版格式

你生成的文章应该在结构粒度和内容深度上与这个范例保持一致。

## 质量检查清单

交付前确认：
- [ ] **目标读者分析完整且具体**（不是"对 AI 感兴趣的人"这种泛泛的描述）
- [ ] 如果原文是英文：翻译准确，专有名词保留英文，术语首次出现有中英对照
- [ ] 如果原文是中文：转写错误已纠正
- [ ] 所有数字、公司名、产品名、时间线准确
- [ ] 受访者的思考深度和表达质感被保留了
- [ ] 每个问题都标注了提问者身份
- [ ] 没有添加受访者没说过的观点或数据
- [ ] **章节数量在 8-12 个之间**（如果少于 8 个，回去检查是否合并过度）
- [ ] **采访者分开讲的话题各自独立成章**
- [ ] **5 个标题基于目标读者分析生成**，不是凭感觉写的
- [ ] 5 个标题覆盖了不同的读者群体和情绪切入点
- [ ] Markdown 语法正确（表格、引用块、标题层级）
- [ ] 文章开头的受访者信息卡片准确
- [ ] **文章总长度在 6000-10000 字之间**（如果明显偏短，检查是否过度压缩）
