# 访谈逐字稿 → 公众号 Markdown 文章

## 背景与定位

这个 Skill 服务于公众号「郭晓文」的内容生产。

**公众号定位：** 面向有野心、不安现状、愿意为自己的野心付出代价的小个体。这些人包括想做副业的上班族、正在做小红书电商的新手、以及所有在寻找适合个体创业的项目并想拿到结果的人。

**内容模式：** 郭晓文（主持人）邀请在小红书或其他适合个体创业领域拿到实际结果的嘉宾进行直播访谈。嘉宾的共同特征是：起点对普通人来说可复制、可参考。内容的核心价值是「此岸是小个体，彼岸是通过小个体可以做的项目拿到结果、实现自己想要的人生状态的人」——文章就是连接两岸的桥。

**为什么用 Q&A 结构：** 访谈天然是问答形式。以主持人的核心问题为骨架来组织内容，比按主题重新归类更真实、更有代入感。读者跟着问题走，就像自己坐在直播间里一样。

## 你的任务

将访谈逐字稿转化为两样东西：

1. **5 个标题选项**（附受众定位和选择理由）
2. **1 篇完整的 Q&A 结构 Markdown 文章**（可直接通过 Markdown 转换器生成公众号排版）

## 工作流程

### 重要原则：宁长勿短，不要压缩

在开始之前，请牢记这条核心原则：**你的默认倾向会是压缩和合并内容，但在这个任务里，这个倾向是错的。**

一场 2-3 小时的访谈，整理后的文章通常在 **6000-10000 字**之间。如果你的输出只有 3000-4000 字，几乎可以肯定你过度压缩了。

为什么不能压缩？因为读者来看这篇文章，就是想看到嘉宾的完整思考过程、具体的操作细节、真实的数据。他们需要的是"我看完就能去做"的信息密度，而不是一个概览。主持人在直播中花时间展开讲的每个话题，都是经过编辑判断的——他选择讲它，说明他认为读者需要它。你的任务是忠实呈现这些内容，不是替读者做"精简"的决定。

### 第一步：通读全文，建立全局理解

仔细阅读完整的逐字稿（可能很长，几万字很正常）。在这一步你需要搞清楚：

1. **谁在说话：** 识别主持人（通常是郭晓文）、联合主持（如李超鹏）、嘉宾、以及嘉宾的搭档/合伙人。注意：有时嘉宾的搭档也会发言，他们的观点同样有价值。
2. **嘉宾是什么人：** 年龄、背景、之前做什么的、现在做什么、核心成绩数据（月利润、GMV、时间线等）。
3. **访谈的主线：** 从哪里聊到哪里？大致的逻辑脉络是什么？
4. **转写错误：** 语音转写软件常见的同音字错误（"小黄书"→"小红书"、"头流"→"投流"、"思域"→"私域"、"生材"→"生财"等），结合上下文判断正确用词。

### 第二步：提取核心问题，构建 Q&A 骨架

这是最关键的一步。你要从几万字的对话中提炼出 **主持人和联合主持提出的核心问题**，这些问题就是文章的骨架。

**提取原则：**

- **保留提问者的身份：** 标注是郭晓文问的还是李超鹏问的（或其他联合主持）
- **合并碎片化的追问：** 直播中主持人可能分好几句话才把一个问题问完，你需要把它们合并成一个完整的、清晰的问题
- **保留追问的层次感：** 有些追问是在深挖同一个话题（比如"利润率多少？"→"为什么后来放弃了？"），这些应该在同一个章节里按顺序呈现，而不是拆成独立章节
- **过滤闲聊：** 直播中有很多过渡性的闲聊（"你吃饭了吗""等一下网络卡了"），这些不需要保留
- **识别观众关心的问题：** 有些问题虽然主持人没有直接问，但嘉宾主动回答了一个观众关心的痛点，也应该纳入

**分章逻辑——这是最容易出错的地方，请特别注意：**

核心原则：**主持人在直播中分开讲的话题，就必须分成独立章节。绝对不要合并。**

具体来说：
- 如果主持人花了 5 分钟专门做基础科普（比如"什么是小红书电商"），这就是一个独立章节。它不是"可以省略的背景"，而是主持人刻意为新读者准备的入口，帮助他们理解后续内容。
- 如果主持人分别用不同的问题去聊选品、笔记制作、店铺运营、投流策略，这些就是 4 个独立章节，不要因为它们都属于"实操方法论"就合并成一个。每个环节都有独立的提问、独立的回答、独立的价值。
- 如果访谈中有一段关于心态/心力/坚持的讨论，即使它穿插在实操内容之间，也应该是独立章节。

判断标准很简单：**在直播中，主持人是不是换了一个新的问题方向？** 如果是，那就是一个新章节。

一场典型的 2-3 小时访谈，整理后通常会有 **8-12 个章节**。如果你只分出了 5-6 个章节，很可能是合并过度了，请回去检查。

参考结构（根据实际内容灵活调整，章节数量不限于此）：
- 嘉宾背景/来时路
- 基础科普（如果主持人专门讲了的话）
- 项目选择/入局过程
- 早期经历（比如第一阶段做了什么品、为什么放弃）
- 心态/心力转变
- 选品方法论
- 笔记制作方法论
- 店铺运营
- 投流策略
- 坚持与日更
- 结语/寄语

每个章节给一个简洁的标题（如"一、来时路：从打工人到创业者"）和一句副标题（斜体，解释这个章节在聊什么）。

### 第二步半：先输出大纲，再逐章展开

在正式写文章之前，先把 Q&A 骨架以大纲形式列出来。格式如下：

```
章节一：[标题]
  - 🎙 郭晓文：[问题1概要]
  - 🎙 郭晓文：[问题2概要]
  - 💡 主持人观点：[关于什么]

章节二：[标题]
  - 🎙 李超鹏：[问题概要]
  - 🎙 郭晓文：[追问概要]
  ...
```

列完大纲后自检：
- 章节数量是否在 8-12 个之间？
- 有没有把本应独立的话题合并了？
- 主持人专门做的科普/铺垫有没有遗漏？

大纲确认无误后，再逐章展开写完整内容。这样做的好处是：每一章都能得到充分的注意力，不会因为文章太长而在后半段开始偷工减料。

### 第三步：填充回答内容

在每个问题下面，整理嘉宾的回答。

**整理原则：**

- **保留口语感：** 嘉宾说"就像饿狼看到肉一样"，保留它，不要改成"就像发现了机会"。口语的生动感是这类内容的生命力。
- **去掉冗余但不去掉风格：** 去掉"呃"、"嗯"、无意义的重复，但保留有表达力的语气词和表达方式。
- **压缩但不失真：** 嘉宾可能用了500字来讲一个观点，你可以压缩到200字，但核心信息和逻辑链不能断。
- **数据必须准确：** 所有数字、百分比、时间、金额都要和原文一致。如果原文自相矛盾，保留更合理的那个。
- **事实核查：** 如果嘉宾或主持人提到某个人物、平台、工具，确保描述准确。如果你不确定某个事实性描述是否正确（比如"XX是某某平台的创始人"），不要擅自加注，原样保留嘉宾的说法即可。

**主持人观点的特殊处理：** 郭晓文经常在嘉宾回答后补充自己的观点和总结，这些内容非常有价值（它是主持人为读者做的"翻译"和"提炼"）。用 💡 标记单独呈现。

**金句提取：** 如果某句话特别有力量、适合单独拎出来（不管是嘉宾说的、主持人说的、还是引用别人的），用引用块单独呈现。

### 第四步：撰写嘉宾介绍卡片

在文章开头，写一段简洁的嘉宾介绍（1-2句话），然后用表格呈现关键数据。

嘉宾介绍的写法：一句话说清楚这个人是谁、做什么的、核心成绩。不要写成简历，要写成「让读者立刻想继续往下看」的钩子。

关键数据表格选 4-6 个最有冲击力的数据点，比如入局时间、当前利润、投入成本、核心指标等。

### 第五步：生成 5 个标题选项

标题是这篇文章能否获得推荐和阅读的关键。

**成功案例参考：** 「【文字版】95后男生，在小红书上卖饰品，月利润10万+」这个标题在公众号获得了2万+阅读和平台推荐。

**标题策略——每个标题要包含以下要素中的至少3个：**

| 要素 | 作用 | 示例 |
|-----|------|------|
| 身份标签 | 让读者产生"跟我差不多"的共鸣 | 95后男生、宝妈、大厂离职 |
| 平台/赛道 | 让读者知道具体在哪里做 | 小红书、抖音、闲鱼 |
| 品类/领域 | 具体到能想象的程度 | 卖饰品、做养生、卖课 |
| 数字化结果 | 可信的、具体的成绩 | 月利润10万+、日出200单 |
| 起点低/反差 | 制造"我也可以"的感觉 | 零基础、投了300块、下班后2小时 |
| 时间线 | 暗示速度和可复制性 | 3个月、半年、从去年6月开始 |

**5 个标题应该覆盖不同的受众切入角度：**

1. **身份共鸣型：** 突出嘉宾身份，让目标读者觉得"这不就是我吗"
2. **结果冲击型：** 用最亮眼的数据打头阵
3. **方法论型：** 暗示文章里有可学的干货
4. **反差/故事型：** 突出起点低→结果好的戏剧性
5. **痛点切入型：** 从读者正在纠结的问题切入

每个标题附上一句话说明：这个标题在打哪个人群、为什么可能有效。

**标题格式建议：** 前面可以加【文字版】或【访谈实录】标签，表明这是直播内容的文字整理，这对已经看过直播的粉丝有召回作用，对新读者有「内容已筛选整理」的信任感。

### 第六步：组装 Markdown 文章

按以下模板组装最终文章：

```markdown
# [正式标题——从5个选项中选一个占位，用户会自己替换]

**[嘉宾名] x [主持人名] x [联合主持名]**
**问答实录 | 核心观点提炼**

---

[嘉宾介绍段落，1-2句话]

| 关键数据 | |
| --- | --- |
| 数据项1 | 数值1 |
| 数据项2 | **数值2**（重要数据加粗） |
| ... | ... |

---

## 一、[章节标题]

*[章节副标题，斜体，说明这一节在聊什么]*

**🎙 [提问者姓名]：[完整的问题]**

> [嘉宾的回答，整理后的版本]
> —— [嘉宾姓名]

**🎙 [提问者姓名]：[追问]**

> [嘉宾的回答]
> —— [嘉宾姓名]

**💡 [主持人姓名]观点：** [主持人的补充观点或总结]

> **[特别有力量的金句，加粗]**
> —— [说话人姓名]

---

## 二、[下一个章节标题]
...

---

## [最后一个章节：结语]
...

---
```

**Markdown 格式要点：**

- `#` 用于文章大标题，`##` 用于章节标题
- 问题用 `**🎙 姓名：问题**` 格式，加粗
- 回答用 `>` 引用块，末尾标注 `—— 姓名`
- 主持人观点用 `**💡 姓名观点：**` 开头
- 独立金句用 `>` 引用块 + `**加粗**`
- 章节间用 `---` 水平线分隔
- 关键数据表格用 Markdown 表格语法
- 需要强调的数字或关键词用 `**加粗**`

### 第七步：输出与交付

最终输出两部分内容，保存为一个 `.md` 文件：

**文件开头：** 5 个标题选项（带简短说明）
**文件主体：** 完整的 Markdown 文章

文件保存到 `/sessions/relaxed-kind-allen/mnt/outputs/[嘉宾姓名]访谈-公众号版.md`

告诉用户：
1. 先从 5 个标题里选一个（或者自己组合改编）
2. 用 Markdown 转换器（如 Md2All、墨滴等）转成公众号格式
3. 复制粘贴到公众号编辑器即可发布

## 参考范例

在 `references/example.md` 中有一篇已发布并获得 2 万+阅读的成品文章。在开始工作之前，先读一遍这个范例，注意以下几点：
- 章节的数量和粒度（10 个章节，每个话题独立）
- 每个章节内问答的深度和细节量
- 主持人科普内容被保留为独立章节
- 整篇文章的总体长度和信息密度

你生成的文章应该在结构粒度和内容深度上与这个范例保持一致。

## 质量检查清单

交付前确认：
- [ ] 转写错误已全部纠正（尤其是同音字）
- [ ] 所有数字、百分比、金额与原文一致
- [ ] 嘉宾的语言风格和生动表达被保留了
- [ ] 每个问题都标注了提问者身份
- [ ] 没有添加嘉宾没说过的观点或数据
- [ ] **章节数量在 8-12 个之间**（如果少于 8 个，回去检查是否合并过度）
- [ ] **主持人专门做的科普/铺垫内容有独立章节**
- [ ] **选品、笔记、运营、投流等实操话题各自独立成章**（如果主持人分开讲了的话）
- [ ] 5 个标题覆盖了不同的受众切入角度
- [ ] Markdown 语法正确（表格、引用块、标题层级）
- [ ] 文章开头的嘉宾数据卡片信息准确
- [ ] **文章总长度在 6000-10000 字之间**（如果明显偏短，检查是否过度压缩）
